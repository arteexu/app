# Tactics Trainer — data source & license

The puzzles in `tactics-trainer.json` are sourced from the **Lichess open puzzle
database**:

- Source: https://database.lichess.org/#puzzles (`lichess_db_puzzle.csv.zst`)
- License: **CC0 1.0 Universal (public domain dedication)** — no attribution is
  legally required; we credit Lichess voluntarily.

## How the curated subset was produced

Generated by `app/scripts/curate-lichess-puzzles.mjs`, which streams the
compressed CSV directly from the official URL (it is never committed), filters
for quality, converts each puzzle into the app's `PuzzleStep` schema, and
validates every line with `chess.js`.

Quality filters (see the script for exact constants):

- Popularity ≥ 90 (community vote score, out of 100)
- NbPlays ≥ 1500 (well battle-tested)
- RatingDeviation ≤ 80 (trustworthy rating)
- Solution ≤ 8 plies (≤ 4 learner moves), and the line must end on the
  learner's move (mate puzzles must actually end in checkmate)

Distribution: 10 motifs × 4 difficulty bands (Beginner 800–1300, Intermediate
1300–1700, Advanced 1700–2100, Expert 2100–2600). Each motif draws a spread
across all four bands and is ordered easy → hard within its lesson.

## Reproduce

```bash
cd app
node scripts/curate-lichess-puzzles.mjs            # re-stream + re-curate
node scripts/validate-tactics-puzzles.mjs          # independent chess.js check
```

The Lichess "Moves" field is UCI, and its **first move is the opponent's setup
move** — the curation applies that move to the FEN so each puzzle starts from
the exact position the learner must solve, then stores the solution in SAN.
